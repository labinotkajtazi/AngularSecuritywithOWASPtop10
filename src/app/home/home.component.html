• The list items below are colored based on the following condition:
  • Issues which fully apply to Angular
  • Issues which do not apply to Angular
  • Issues which partially apply to Angular

  • -Injection-
    Includes all injection types such as SQL Injection, Template Injection etc
  • -XSS-
    Cross Site Scripting - refers to client-side code injection attack wherein an attacker can execute malicious scripts into a legitimate website or web application
  • -BASM-
    Broken Authentication and Session Management - Can occur when user credentials are weak or not properly stored or when session management is flawed
  • -IDOR-
    Insecure Direct Object References - Insecure Direct Object References occur when an application provides direct access to objects based on user-supplied input. As a result of this vulnerability attackers can bypass authorization and access resources in the system directly, for example database records or files
  • -SM-
    Security Misconfiguration - Typically occur when holes are left in the security framework of an application by systems administrators, DBAs or developers
  • -SDE-
    Sensitive Data Exposure - Sensitive Data Exposure occurs when an application does not adequately protect sensitive information such as passwords, session tokens, credit card data to private health data etc
  • -MFLAC-
    Missing Function Level Access Control - Occurs if user authentication in sensitive request handlers is insufficient or non-existent, therefore leaving a vulnerability
  • -CSRF-
    Cross-Site Request Forgery - An attack that forces an end user to execute unwanted actions on a web application in which they're currently authenticated. CSRF attacks specifically target state-changing requests, not theft of data, since the attacker has no way to see the response to the forged request
  • -UCKV-
    Using Components with Known Vulnerabilities - A threat identified through vulnerabilities in third-party libraries and software which are extremely common and could be used to compromise the security of systems using the software
  • -URF-
    Unvalidated Redirects and Forwards - Unvalidated redirects and forwards are possible when a web application accepts untrusted input that could cause the web application to redirect the request to a URL contained within untrusted input
