-Unvalidated Redirects and Forwards-

If you have created a web application chances are there are going to be links redirecting to other pages or other areas within the same page. This leaves space for unwanted action from attackers, such as changing the link and redirecting users to malicious websites etc. In order to prevent this behaviour it falls on the developers to make sure they do not use actual URL's but instead an ID or other value which is mapped and translated in the back so that the redirect can only be within a finite number of trusted URL's.


-Code Responsible-

        Wrong - <a href="http://localhost:4200/home">Redirect to Home</a>
        

        Right - 
        HTML part
        <a [routerLink] ="[getLink('2')]">Redirect to Home</a>

        Typescript part
        links = [
           { id:'1', value: '../uckv' },
           { id:'2', value:'../home' }
        ];
    
        getLink(url: string) {
          return (this.links.find(i => i.id === url)).value;
        }
