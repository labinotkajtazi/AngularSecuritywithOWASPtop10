-Using Components with Known Vulnerabilites-

When building applications developers require certain components which make everything easier and more user friendly. Unfortunately that often leads to components with known vulnerabilites which in turn leaves the application with security holes. Angular cannot provide protection against these types of threats so it falls on the developers shoulders to analyse the components they are going to use in detail before including them to the application and exposing the application. There are different ways of analysing such components, some of which are

  • Review other users comments on the specific library
  • Use different detection tools, examples (Snyk, RetireJs, Burp etc)
  • Reviewing the code by yourself
